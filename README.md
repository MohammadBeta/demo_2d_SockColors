# 2D Sock Customizer

React 18 sock customization app using Fabric.js on a 2D canvas. This project intentionally does not use Three.js or WebGL.

## Features

- Loads the sample `public/assets/shoe.png`, or lets the user upload a PNG, JPG, WebP, or other browser-supported image.
- Fits uploaded images automatically into the canvas, including very wide, very tall, small, or large files.
- Recolors dark and light products using an alpha/luminance mask generated with Canvas 2D. This avoids the black-image problem that happens with a plain `multiply` overlay.
- Includes three transparent PNG logo assets:
  - `public/assets/logo-spark.png`
  - `public/assets/logo-bolt.png`
  - `public/assets/logo-wave.png`
- Logos can be clicked or dragged onto the canvas.
- Added logo objects are Fabric.js objects, so they can be dragged, scaled, and rotated with canvas handles.
- Saves the finished design as a PNG using `canvas.toDataURL()`.
- Styled with Tailwind CSS.

## Components

- `src/components/ShoeCanvas.jsx` - owns the Fabric.js canvas, uploaded product image, recoloring mask, logo objects, drag/drop, and PNG export.
- `src/components/ProductImagePicker.jsx` - lets users choose their own sock/product image.
- `src/components/ColorPicker.jsx` - color presets, native color picker, and color strength slider.
- `src/components/LogoSelector.jsx` - selectable and draggable logo picker.
- `src/components/SaveDesign.jsx` - PNG download button.

## Getting Started

Install dependencies:

```bash
npm install
```

Run the dev server:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

Preview the production build:

```bash
npm run preview
```

## Regenerating Sample Assets

The included PNG assets are generated by a PowerShell script:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\generate-sample-assets.ps1
```

You can replace any files in `public/assets` with your own transparent PNG sock/product or logo artwork. Users can also upload their own images directly in the app.

## Notes

- The canvas export uses a `multiplier` of `2` for a sharper downloaded PNG.
- Uploaded images are processed with Canvas 2D and capped internally at `1800px` on the longest side for smoother browser performance.
- Transparent PNGs give the cleanest product outline. JPGs and images without transparency still work, but their whole rectangular background will be recolored too.
- npm may report audit findings from Fabric.js transitive dependencies. Do not run `npm audit fix --force` casually because it can upgrade across breaking major versions.
